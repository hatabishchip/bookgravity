# Метапромт: переключатель EN/ID для просмотра черновика агента тренером

Утверждён кнопкой: Edit/Send ВСЕГДА берут английский оригинал; Bahasa - только для чтения.
Охват: карточка предложения агента (появляется только в Canggu - агент-студия = {canggu}).

## Цель (слова владельца 22.07)

Тренеры Canggu - индонезийцы. Черновик агента теперь всегда на английском (метапромт
[[META_agent_english_draft_client_language]]). Дать тренеру возможность ПОСМОТРЕТЬ черновик на
Bahasa, чтобы понять суть - и переключаться обратно на английский. Рядом с кнопками Edit и Send
поставить переключатель из двух радиокнопок English / Bahasa (нельзя выбрать обе, как тумблер).

**ЖЁСТКО: это ТОЛЬКО отображение в мессенджере тренера. Ни на что не влияет.** Что бы тренер ни
выбрал (English или Bahasa), при Send/Edit используется английский оригинал, а клиенту сообщение
уходит на ЕГО языке (перевод на отправке, уже работает). Переключатель не меняет ни отправляемый
текст, ни язык доставки, ни какие-либо данные - только текст, который тренер видит в карточке.

## Что делаем

### 1. API-роут `app/api/whatsapp/translate-preview/route.ts` (POST)
- Auth: `requireAuth` (админ и тренер - как в messages route).
- Тело (zod): `{ text: string, targetLang: string }`.
- Логика: `translateAndDetect({ text, targetLang })`; вернуть `{ translated }`. При неудаче -
  `{ translated: text }` (fallback = показать оригинал), 200.
- ТОЛЬКО перевод для предпросмотра. НИКАКИХ записей в БД, НИКАКИХ отправок. Идемпотентно, read-only.

### 2. Карточка предложения в `app/_components/Inbox.tsx` (~2287)
- Рядом с Edit/Send добавить мини-тумблер `[ EN | ID ]` (радио-стиль, активна ровно одна).
- Состояние в компоненте: `previewLang: "en" | "id"` (по умолчанию **"en"**), `bahasaText: string | null` (кэш).
- Рендер текста черновика (сейчас `{detail.suggestion.draft}`, ~2293):
  - `previewLang==="en"` -> показываем `suggestion.draft` (английский оригинал).
  - `previewLang==="id"` -> если `bahasaText` пуст, POST `/api/whatsapp/translate-preview`
    `{ text: suggestion.draft, targetLang: "id" }`, сохранить в `bahasaText`, показать; пока
    грузится - «…». Повторное переключение берёт из кэша (без повторного запроса).
- При смене предложения (новый `suggestion.id`) - сбросить `previewLang="en"` и `bahasaText=null`.
- **Edit (~946-950) и Send - НЕ трогаем.** Оба используют `suggestion.draft` (английский). Тумблер
  на них не влияет вообще.

## Что НЕ меняется
- Отправляемый текст = английский оригинал -> авто-перевод в язык клиента на отправке.
- Логика агента, промпт, inboxLanguage - без изменений.
- Другие студии (агента нет - карточки нет).
- Никаких схема-миграций (перевод on-demand, ничего не храним).

## Проверка (E2E)
1. Карточка агента в Canggu: виден тумблер EN/ID рядом с Edit/Send; по умолчанию EN.
2. Нажать ID -> текст черновика показывается на Bahasa; нажать EN -> снова английский. Переключение
   мгновенно после первого перевода (кэш).
3. При ID нажать Send -> в чат уходит английский оригинал, переведённый в язык клиента (проверить
   что клиент получил на своём языке, а не на Bahasa). Edit при ID -> в композер английский.
4. Радио: одновременно активна только одна кнопка.
