# Метапромт: переключатель Original/ID на сообщениях клиента для тренеров Canggu

Утверждено кнопкой: показывать ТОЛЬКО тренерам (role TRAINER). Админ - без изменений.

## Цель (слова владельца 22.07)

Тренеры Canggu - индонезийцы. Сейчас (Inbox.tsx:531-537) авто-перевод и кнопка «Translate»
показываются ТОЛЬКО админу; тренер видит входящее сообщение клиента как есть, на языке клиента,
без перевода - и вынужден идти во внешний переводчик. Дать тренеру прямо на сообщении клиента
переключатель **Original / ID** (как у карточки агента EN/ID), чтобы он сразу прочитал, что
написал клиент, на Bahasa. Кнопки на самом сообщении, подписи «Original» и «ID» (не слово
«Translate»).

## Что делаем

Область: `role === "TRAINER"` И студия = **Canggu** И сообщение ВХОДЯЩЕЕ (`direction === "INBOUND"`)
И есть текст (`m.body`). Только эти сообщения получают тумблер. Исходящие, картинки/аудио без
текста, другие студии, админ - не трогаем.

Поведение:
- По умолчанию активна **Original** - показывается `m.body` (оригинал клиента), как сейчас.
- **ID** - по запросу переводит `m.body` на Bahasa через существующий роут
  `/api/whatsapp/translate-preview` `{ text: m.body, targetLang: "id" }`, показывает перевод; при
  загрузке «…». Кэш в состоянии сообщения (один запрос на сообщение). Назад на Original - мгновенно.
- Радио-стиль (активна одна), маленькие кнопки под текстом пузыря, как EN/ID на карточке агента.

## Реализация
- `app/trainer/inbox/page.tsx`: в запрос студии добавить `slug`, передать `<Inbox role="TRAINER"
  studioSlug={studio.slug} />`.
- `app/_components/Inbox.tsx`:
  - Компонент `Inbox`: принять опциональный проп `studioSlug?: string`. Вычислить
    `bahasaToggle = role === "TRAINER" && studioSlug === "canggu"`.
  - В `MessageBubble` передать проп `showBahasaToggle: boolean` = `bahasaToggle && m.direction ===
    "INBOUND" && !!m.body`.
  - В `MessageBubble`: локальное состояние `bahasaShown: boolean` (default false),
    `bahasaText: string | null`, `bahasaLoading: boolean`. Когда `showBahasaToggle` - под текстом
    рендерить радио `Original | ID`; `primaryText` для показа = `bahasaShown ? (bahasaText ?? "…")
    : m.body`. При выборе ID - fetch translate-preview (если не в кэше).
- Роут `translate-preview` уже есть (auth = requireAuth, покрывает и тренера). Ничего в БД не
  пишет, ничего не шлёт.

## Что НЕ меняется
- Админ: авто-перевод на inboxLanguage (EN) + кнопка Translate - как сейчас.
- Отправка сообщений, логика агента, inboxLanguage - без изменений.
- Другие студии, исходящие сообщения.

## Проверка (E2E)
1. Залогиниться ТРЕНЕРОМ Canggu, открыть чат с входящим не-Bahasa сообщением клиента.
2. Под сообщением клиента виден тумблер Original | ID; по умолчанию Original (текст клиента).
3. Нажать ID -> текст сообщения показывается на Bahasa; назад Original -> оригинал, мгновенно.
4. Тумблер есть только на входящих; на исходящих (наших) его нет.
5. Зайти АДМИНОМ - у него по-прежнему авто-EN + Translate, тумблера Original/ID нет.
