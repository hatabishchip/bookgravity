# МЕТАПРОМТ v2: Агент-автопилот - надёжность 15 мин, FAQ-переработка, пуш тренеру, выделение в UI

Версия 2 (продолжение docs/META_agent_autopilot.md). Дата: 16.07.2026. Утверждён владельцем: (ожидает).

## Решения владельца (кнопками, 16.07)
- Проверка чатов каждые **15 минут** (было 30).
- Агент МОЖЕТ отвечать на **общие** вопросы о расписании и ценах (часы занятий, стоимость, как записаться + ссылка). Конкретная запись/перенос/отмена/«я приду сегодня» - по-прежнему тренеру.
- При BOOKING/ESCALATE-пометке агент **шлёт WhatsApp тренеру**: «Клиент X ждёт: <суть>». 1 уведомление на подсказку, дедуп.
- Дневная сводка - НЕ нужна.

## Фаза 1. Надёжность механизма (чтобы «чётко работало»)
1. Hermes crontab: `*/30` -> `*/15` (лог с timestamp уже настроен).
2. Самодиагностика: если sweep падает (не-200) 3 раза подряд - Hermes шлёт владельцу пинг в @hatabish_bot (личный канал, без одобрений). Скрипт-обёртка на сервере вместо голого curl: пишет статус, считает подряд-ошибки.
3. Гард дублей: перед отправкой повторно проверять «последнее сообщение всё ещё INBOUND» непосредственно перед send (окно 15 мин уменьшает гонку с тренером, но проверка обязательна - уже есть, подтвердить).

## Фаза 2. Переработка знаний (FAQ) + границы
1. Собрать ПОЛНЫЙ FAQ Чангу из реальных источников: сайт (StudioInfo), БД (цены, типы занятий: группа/приват/дети, абонементы, типичные часы слотов), уже одобренные факты. Черновик FAQ (EN, голос Андрея) - **показать владельцу на подтверждение фактов** (душ? парковка? что взять с собой? детские занятия? приваты и их цена?). Только подтверждённое идёт в промт.
2. Новая классификация:
   - SAFE+ (агент отвечает): что это, цена группы, длительность 75-90, до 6 человек, общие часы работы/дни, как записаться (ссылка), локация, что взять/во что одеться, уровни/возраст, IG.
   - BOOKING (тренеру): конкретная дата/время/запись/перенос/отмена/опоздание/«я у двери»/помощь с бронированием/друг тоже хочет.
   - ESCALATE (тренеру/владельцу): деньги, жалобы, медицина, партнёрства, вакансии.
3. Пересмотреть уроки (10 в БД): дубли слить, слабые выключить.
4. Качество генерации: 2 few-shot эталонных диалога в промт (новый лид + возвращающийся клиент); признак «новый лид vs клиент с бронями» (есть ли брони у телефона) - тон меняется; лимит длины ответа; гард «не отвечать на вопросы не про студию».
5. Тест-прогон БЕЗ отправки: 12 типовых вопросов (EN/RU/ID) через реальный пайплайн, самопроверка + показать владельцу таблицей «вопрос -> ответ агента» на финальную приёмку.

## Фаза 3. Пуш тренеру при «нужен человек»
1. Поле `trainerNotifiedAt` на AgentSuggestion (миграция).
2. В sweep: для pending BOOKING/ESCALATE без trainerNotifiedAt - отправить WhatsApp тренеру (assignedTrainer разговора; если нет - тренеры студии с notifyWhatsapp=true) текстом: «Client <имя> is waiting in the inbox: <reason>. Open: bookgravity.com/admin/inbox». Пометить trainerNotifiedAt.
3. Лимиты: максимум 1 уведомление на подсказку; не слать ночью Бали (23:00-07:00) - утром догонит; если тренер ответил в чате до sweep - не слать.

## Фаза 4. Выделение агента в UI (для тренера/админа)
1. Пузырь от агента: фиолетовый тинт + подпись «Agent 🤖» (подпись есть - добавить тинт).
2. Список чатов: бейдж 🤖 на карточке, если ПОСЛЕДНИЙ исходящий - от агента (тренер сразу видит «этим занимался агент»).
3. Карточка «needs your reply» (жёлтая пометка): добавить кнопку «Ответлено голосом» нет - НЕ усложнять; оставить Send/Edit/Dismiss.
4. UK-ключи для новых строк.

## Критерии готовности
- Крон 15 мин подтверждён в syslog; 3 фейла подряд = пинг владельцу (проверить симуляцией).
- FAQ подтверждён владельцем, вшит; тест-прогон 12 вопросов принят владельцем.
- Пуш тренеру: E2E на тестовом BOOKING (Sancho) - тренеру пришло WhatsApp, второй раз не шлётся.
- Тинт+бейдж видны в превью, скриншот владельцу.
