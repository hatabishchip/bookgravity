# META: медиа в чате - без моргания + прогресс загрузки (утв. 15.07.2026)

Только фронт (`app/_components/Inbox.tsx`), сервер и маршруты не трогать.

## Цель
Фото/видео в чате показывается СРАЗУ и не мигает; поверх - индикатор загрузки (реальные % на фазе аплоада телефон->сервер, кружок на фазе сервер->Meta); если обработка тянется >8с - подпись «Still sending…» под медиа.

## Причина моргания
После ответа сервера src переключается с локального blob: на прокси /api/whatsapp/media/[id], а blob сразу revoke'ается -> пустой кадр пока прокси грузится.

## Реализация
1. Убрать моргание: blob-URL держать в кэше id->url, показывать после ответа сервера; preload прокси скрытым Image(); переключать src на прокси только по onload, тогда revokeObjectURL. Видео - тот же blob до готовности прокси.
2. Прогресс: multipart (мелкие) -> XMLHttpRequest с upload.onprogress; Blob (крупные видео) -> onUploadProgress @vercel/blob/client. Поля на оптимистичном пузыре: uploadPct (0-100) + phase ('uploading'|'processing'), throttle до целых %.
3. Overlay (стиль WhatsApp): тёмный слой + круговой индикатор; uploading = число NN%, processing = спиннер. Зум заблокирован пока грузится.
4. «Still sending…» под медиа когда phase=processing длится >8с. Ошибка -> прежний «Not delivered» + повтор.
5. Все blob revoke после preload прокси или при размонтировании; кэш чистить.

## Проверка
E2E превью: фото + видео реальной ролью, % на аплоаде, кружок на обработке, ноль моргания финальной картинки, консоль без утечек. Затем деплой.

## Критерии
Прогресс апдейтит только свой пузырь; отправка/повтор/ошибки не ломаются; клиентские тексты и серверные пути не трогаются.
