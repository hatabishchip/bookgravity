# МЕТАПРОМТ: Автопилот sales-агента (Чангу) + журнал RU + обучение на Убуде + разбор неотвеченных

Версия 1. Утверждён владельцем: (ожидает подтверждения). Дата: 15.07.2026.
Продолжение фазы 1 агента (suggest-режим, live с 15.07). Спека фазы 1: lib/sales-agent.ts + agent/sales-agent-knowledge.md.

## Решения владельца (кнопками, 15.07)
- Авто-отправка: ТОЛЬКО SAFE. BOOKING/ESCALATE остаются подсказкой тренеру с жёлтой пометкой (как сейчас).
- Частота: sweep каждые 30 минут (Vercel cron). Побочный плюс: у тренера есть до 30 мин перехватить черновик до авто-отправки.
- Разбор неотвеченных: вся история.
- Массовая отправка шаблона в закрытые неотвеченные: полная автономия (без показа списка).

## Цель
Агент сам отвечает клиентам Чангу на SAFE-вопросы; владелец видит журнал «что спросили - что ответили» на русском в админке; агент постоянно учится (правки тренеров, отклонения, история Убуда); все исторически неотвеченные чаты получают ответ (открытое окно - живой ответ агента, закрытое - временный Meta-шаблон, который после рассылки удаляется).

## Фаза A. Самообучение (каркас) - ДО автопилота, чтобы автопилот сразу писал уроки
1. Таблица `AgentLesson` (id, createdAt, source: edited_sent|dismissed|ubud_history|owner, lesson text, active bool). Миграция prod Turso напрямую через @libsql/client (рецепт из хендовера, проверять PRAGMA table_info).
2. KNOWLEDGE собирается на лету: базовые знания (lib/sales-agent.ts) + активные уроки из БД. Урок в БД = без деплоя.
3. Извлечение уроков: в cron-обработчике (фаза B) - свежие edited_sent (diff черновик vs отправленное) и dismissed → Gemini извлекает дельту → INSERT AgentLesson (дедуп по смыслу: передавать существующие уроки в промт «не дублируй»).
4. Просмотр/выключение уроков: блок «Уроки» на /admin/agent-stats (список, тумблер active). UK-ключи в uk.ts.

## Фаза B. Автопилот SAFE + cron sweep 30 мин
1. `app/api/cron/agent-autopilot/route.ts` + `vercel.json` crons `*/30 * * * *` (защита CRON_SECRET).
2. Логика sweep (только studio slug=canggu, гейт сохраняется):
   - Найти разговоры, где последнее сообщение = inbound от клиента и после него нет исходящего.
   - Если нет pending suggestion - сгенерировать (generateAgentSuggestion; добирает пропуски вебхука).
   - Для pending SAFE: отправить от агента (fromAgent=true, suggestion → sent). Окно открыто - free-form; окно закрыто - существующая admin_message-обёртка.
   - BOOKING/ESCALATE - НЕ отправлять, остаются пометкой тренеру.
   - Защиты: не слать, если после входящего уже есть ЛЮБОЙ наш ответ; один авто-ответ на одно входящее (статус suggestion); максимум 1 авто-ответ подряд - если клиент не ответил на авто-ответ, повторно не писать; лимит 30 отправок за sweep (защита от лавины).
3. Извлечение уроков (фаза A п.3) выполняется в конце того же sweep.

## Фаза C. Журнал ответов в админке (RU)
1. Поля в AgentSuggestion: `questionRu`, `answerRu` (перевод Gemini в момент отправки/правки; для истории - бэкфилл).
2. Страница `/admin/agent-log` (пункт меню рядом с «Агент», только Чангу): таблица Время | Клиент | Что спросили (RU) | Что ответили (RU) | Как (авто / тренер с правкой / тренер как есть / отклонено). Пагинация, свежие сверху. UK-ключи.
3. Мобильная вёрстка (владелец смотрит с телефона).

## Фаза D. Обучение на истории Убуда (разово)
1. Прочитать ВСЕ WhatsApp-чаты студии Убуд из prod-БД (обе стороны, хронологически).
2. LLM-анализ пар «вопрос клиента → ответ студии → что дальше»: успех = клиент записался/пришёл/ответил позитивно; неуспех = замолчал/отказался.
3. Извлечь уроки (формулировки, скорость, структура, ошибки) → AgentLesson (source=ubud_history) + зеркало в agent/sales-agent-knowledge.md.
4. Отчёт владельцу: топ-паттерны успеха/провала одной страницей в чат.

## Фаза E. Разбор всех неотвеченных чатов (разово, вся история, обе студии)
1. SQL: разговоры, где последнее сообщение inbound и без нашего ответа (вся история).
2. Окно открыто (<24ч): агент отвечает обычным пайплайном (SAFE - сам; BOOKING/ESCALATE - пометка тренеру).
3. Окно закрыто: временный Meta-шаблон (EN, без переменных, голос Андрея, только дефисы), пример текста:
   «Hi! This is Gravity Stretching. Sorry we missed your message earlier - we are here now and happy to help. Just reply and we will answer right away 🙏»
   Подать в Meta → дождаться approve (вотчер) → разослать по списку закрытых неотвеченных → УДАЛИТЬ шаблон из Meta (приказ владельца: не засорять шаблоны).
4. Ответившие попадают в обычный поток (окно откроется, агент/тренер продолжат).
5. Отчёт: сколько чатов найдено, сколько отвечено живым текстом, сколько шаблоном, сколько ответили.

## Допущения (явно)
- Автопилот и журнал - только Чангу (гейт агента). Разбор неотвеченных (фаза E) - ОБЕ студии («проверь все чаты»); в Убуде авто-ответов нет, там только шаблон/живой ответ без агента... нет: фаза E для Убуда шлёт тот же reconnect-шаблон, а дальше отвечает тренер Убуда (агент не включается).
- «Совершенствуйся» = фаза A (уроки из правок/отклонений) + фаза D (Убуд).
- Владелец журнала: только просмотр; управление агентом - на agent-stats.
- Жёсткие границы агента не меняются (медицина/деньги/жалобы = эскалация).

## Критерии готовности
- Cron виден в Vercel, sweep проходит без ошибок, SAFE-ответ реально уходит клиенту (E2E на бот-чате Sancho).
- Журнал показывает вопрос/ответ на русском, статусы верные.
- Уроки появляются из правки тренера (E2E: правка → урок в БД → виден на agent-stats).
- Убуд-уроки записаны, отчёт отдан.
- Неотвеченные обработаны, шаблон удалён из Meta, отчёт отдан.
