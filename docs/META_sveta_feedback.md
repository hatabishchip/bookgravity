# Метапромт: фидбек Светы (Cancel в кабинете + прозрачность зарплаты)

Источник: голосовые/скрины Светы (сооснователь, жена Андрея) + тренер Ячинта, 21.06.2026.
Решения по развилкам подтверждены владельцем кнопками.

## Цель
Три правки в bookgravity по фидбеку Светы:
- A. Кнопка **Cancel** в кабинете тренера (вместо No-show).
- B. Убрать визуальную путаницу «двух кнопок» Reschedule.
- C. Полная **прозрачность зарплаты**: разбивка по классам у тренера (read-only) и у админа (для сверки).

---

## A. Cancel вместо No-show (деталь класса в "My Schedule")
Файл: `app/trainer/page.tsx` (+ зачистка NO_SHOW по коду).

- **Убрать No-show** из детали класса: кнопки markNoShow/undoNoShow и рендер статуса NO_SHOW.
- **Убрать списание абонемента при неявке** (NO_SHOW-burn) - пока не нужно.
- **Добавить кнопку "Cancel"** в карточку клиента - та же механика, что уже работает в "All My Bookings":
  статус CANCELLED, абонемент **возвращается** (не списывается), бронь уходит из начисления зарплаты,
  клиент перестаёт висеть в "собрать оплату".
- Перед отменой - подтверждение ("Cancel this client?"), чтобы не отменить случайно.
- Уведомление клиента при отмене - **повторить текущее поведение** cancel из /trainer/bookings (консистентность).
- Зачистка NO_SHOW по коду: `lib/payments.ts` BOOKING_STATUSES → `["CONFIRMED","CANCELLED"]`;
  roster-фильтры `{in:["CONFIRMED","NO_SHOW"]}` → `"CONFIRMED"`; UI admin/bookings и trainer/bookings.
  Колонку `Booking.status` оставить (аддитивно), просто перестать выставлять NO_SHOW.

## B. Reschedule - один понятный контрол + Cancel рядом
Файл: `app/trainer/page.tsx`.

- Перенос = ОДИН контрол (кнопка "Reschedule" раскрывает выбор класса), без отдельного дубль-вида
  "RESCHEDULE" (заголовок) + "Move to another class" (ссылка).
- Рядом - отдельная кнопка "Cancel" (из пункта A). Теперь это две РАЗНЫЕ функции, не дубль.

## C. Прозрачность зарплаты - разбивка по классам

### Тренер
Файлы: `app/api/trainer/salary/route.ts`, `app/trainer/salary/page.tsx`.

- API: добавить массив `breakdown` - по каждой оплаченной брони за месяц:
  дата, имя клиента, тип класса, сумма оплаты, способ оплаты, комиссия тренера
  (с учётом local-цены и assistant-rate).
- UI: под итогом - список "за что и сколько" (read-only). Тренер видит все N классов
  и сколько каждый принёс ему. Снять путаницу "1М оклад": явно показать, что это комиссия 20%
  с оплаченных броней, базового оклада нет (`baseSalary:0` подтверждён кодом).

### Админ
Файлы: `app/api/admin/salary/route.ts`, `app/admin/salary/page.tsx`.

- API: по каждому тренеру - тот же `breakdown` (дата, клиент, класс, сумма, способ оплаты, комиссия).
- UI: раскрывающаяся разбивка под каждым тренером (как таблица Светы "Employee commissions":
  можно сверить каждый класс - был ли, как оплачен, сколько комиссии). Итоги сходятся с суммой строк.

## Ограничения / критерии успеха
- Без длинного тире нигде в UI (только дефис "-").
- Изменения схемы (если нужны) - аддитивно через libsql ALTER, прод не ломать.
- Проверка локально на изолированной БД (тренер gr@gr.gr) ДО деплоя:
  Cancel возвращает абонемент + убирает из зарплаты; breakdown сходится с итогом у тренера и админа.
- Коммит+пуш после; VersionWatcher подхватит без ручного сброса кэша.
- Зарплатная модель: flat 20%, базового оклада нет.

## Развилки (подтверждено)
1. No-show → заменить на Cancel, списание абонемента при неявке убрать.
2. Reschedule оставить (один контрол) + Cancel рядом.
3. Разбивка - полная (как у Светы): тренер свою read-only, админ по каждому тренеру.
