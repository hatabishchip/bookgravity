# META Part_7: деплой-фикс (функции < 250МБ) + доводка клавиатуры чата

Утверждено владельцем 15.07.2026 (v1). Ответы: кастомную клавиатуру оставить и довести (полный пакет), деплой приземляет все накопленные коммиты и убирает бету large-functions.

## Цель
1. Прод деплоится обычным путём: каждая serverless-функция < 250МБ, без `VERCEL_SUPPORT_LARGE_FUNCTIONS`; все накопленные коммиты live.
2. Клавиатура чата (Composer + VirtualKeyboard + жестовый движок Inbox) доведена: удобство набора, логичность, один архитектурный владелец состояния.

## Этап 1 - деплой-блокер (первым)
1. Диагностика по факту: локальный build, анализ traced-размера функции (.nft.json / .vercel/output), найти что даёт 317МБ. Не гадать.
2. Фикс по данным, приоритет:
   - (a) вывести prisma-стек из external (Prisma 7 driver adapter = JS+wasm, бандлится; при необходимости transpilePackages для отмены дефолтного external-списка Next);
   - (b) если (a) упрётся - точечный outputFileTracingExcludes строго по отчёту;
   - (c) проверить, не тянется ли CLI-пакет `prisma` в прод-трейс.
3. Валидация: фактический замер размера функций < 250МБ; `npm run build` зелёный.
4. Деплой: push в main, poll /api/version до HEAD-sha, smoke-job зелёный. (Подтверждён владельцем.)
5. Уборка: убрать `VERCEL_SUPPORT_LARGE_FUNCTIONS` и `VERCEL_ANALYZE_BUILD_OUTPUT` из env, отключить вотчер `bookgravity-deploy-retry`.

Критерий успеха: bookgravity.com отдаёт новый HEAD-sha, деплой прошёл без беты.

## Этап 2 - клавиатура (полный пакет)
1. Архитектура (поведение 1:1, механику скрытия НЕ менять - правило владельца): хук `useKeyboardSheet` (keyboardOpen + --kb-h/--kb-off + жестовый движок) + `KeyboardShell`; один писатель --kb-off; Inbox худеет.
2. Набор: long-press альтернативы с пузырём (ь -> ъ, е -> ё, допзнаки в символах); автозаглавная (начало сообщения, после «.!? »); двойной пробел = «. »; Caps Lock двойным тапом; второй символьный слой #+= (% * _ « » € и пр. по iOS).
3. Логика: язык клавиатуры запоминается (localStorage, дефолт по роли = fallback); эмодзи-кнопка отвязана от onAttach; стикер-переключатель включён и тренеру (допущение утверждено).
4. Диктовка: interim-текст виден; нет SpeechRecognition - кнопка микрофона скрыта (без alert); проверить на устройстве.
5. E2E: preview + логин админом и тренером, проверить ъ/ё, автозаглавную, символы, драг, стикеры; деплой (второй и последний).

Критерии качества: ноль новых ре-рендеров на клавишу (uncontrolled сохранён), латентность не хуже, драг-поведение неизменно, клиентские тексты/шаблоны не трогаются.
