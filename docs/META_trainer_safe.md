# МЕТАПРОМТ: «Сейф тренера» - учёт наличных в студии по тренерам

Версия: 1.0 (черновик, 06.07.2026). Статус: ожидает утверждения владельцем.
Решения владельца (кнопками, 06.07): приход = ВСЕ cash-деньги; атрибуция = кто
отметил оплату; операции = изъятие + зарплата из сейфа + коррекция; старт =
ретро-пересчёт по всей истории.

## 1. Цель

Каждый тренер складывает полученную наличку в свой сейф. Система в любой момент
показывает: админу - сколько наличных лежит в сейфе каждого тренера; тренеру -
сколько в его сейфе (цифра должна сходиться с реальной пачкой денег). Владелец
при выплате зарплаты видит, что её можно выдать из сейфа тренера, и сколько
останется забрать при визите в студию. Фича опциональная: тумблер в /sadmin,
по умолчанию ВЫКЛ; после обкатки включается для Чангу.

## 2. Модель учёта (гибрид: приход считается, расход записывается)

**Баланс сейфа тренера = вычисленный cash-приход - записанные расходы.**

- **Приход (автоматически, не дублируется в отдельные строки):**
  - разовые классы: Booking PAID + paymentType CASH (цена по priceTier/localPrice);
  - абонементы за наличные: Membership paymentType CASH (classPrice x totalClasses);
  - доп-услуги наличными (по оплате брони).
- **Атрибуция прихода:** новое поле `Booking.paymentMarkedByUserId` (кто нажал
  «Paid - Cash», тот и положил в сейф). Для Membership уже есть `soldByUserId`.
  ИСТОРИЯ (до этого поля): фолбэк = ведущий тренер слота. Расхождения от
  фолбэка выправляются коррекцией при первом пересчёте сейфов в студии.
- **Расход (ручные записи, новая модель `SafeOperation`):**
  - `withdrawal` - «владелец/админ забрал из сейфа X» (кнопка у админа);
  - `salary` - выплата зарплаты из сейфа: в модале выплаты на /admin/salary
    появляется галочка «Pay from trainer's safe» - создаёт операцию на сумму
    выплаты, привязана к TrainerPayment;
  - `correction` - «пересчитали, фактически в сейфе Y» (знак любой, обязательная
    причина). Ей же вносится стартовая сверка после ретро-пересчёта.

## 3. Схема (аддитивно)

- `Studio.safeEnabled Boolean @default(false)` - тумблер супер-админа.
- `Booking.paymentMarkedByUserId String?` - пишется при каждой отметке оплаты
  (trainer/admin PATCH), независимо от способа оплаты (пригодится и для аудита).
- Модель `SafeOperation`: id, studioId, trainerId, kind ("withdrawal" | "salary"
  | "correction"), amount (Float, знак: минус = из сейфа), note, paymentId?
  (TrainerPayment при kind=salary), createdByUserId, createdAt. Индексы
  [studioId, trainerId], [createdAt].

## 4. API

- `GET /api/admin/safe` - балансы по всем тренерам студии + история операций
  (+ разбивка прихода за период). 403 если !safeEnabled.
- `POST /api/admin/safe/operations` - создать withdrawal/correction (zod,
  requireAdmin, сумма > 0 для withdrawal, note обязателен для correction).
- Выплата из сейфа: расширение существующего POST выплаты зарплаты
  (`/api/admin/salary/payments`) флагом `fromSafe: true` → создаёт SafeOperation
  kind=salary в той же транзакции.
- `GET /api/trainer/safe` - баланс и история СВОЕГО сейфа (requireTrainer).
- Балансы считает общий `lib/safe.ts`: `computeSafeBalances(studioId)` -
  один проход по cash-оплатам (атрибуция: paymentMarkedByUserId → фолбэк
  тренер слота; membership: soldByUserId) минус сумма операций.

## 5. UI

- **/sadmin**: тумблер «Cash safe tracking» на карточке студии (как whatsappEnabled).
- **Админ**: страница `/admin/safe` («Safes» в сайдбаре, виден только при
  safeEnabled): карточка на тренера - баланс крупно, «в т.ч. приход за месяц»,
  кнопки «Withdraw» и «Correction», история операций; итого по студии сверху.
- **/admin/salary**: в модале выплаты галочка «Pay from trainer's safe
  (Rp X available)» - активна при safeEnabled и балансе > 0.
- **Тренер**: карточка «Cash in my safe» в кабинете (рядом с зарплатной):
  баланс + последние операции + подпись «должно совпадать с наличными в твоём
  сейфе - расхождение сообщи админу».
- Валюта - валюта студии (formatIDR / currency).

## 6. Ретро-пересчёт (выбор владельца) и включение для Чангу

1. Построить балансы по ВСЕЙ истории cash-оплат (атрибуция-фолбэк: тренер слота).
2. Показать владельцу таблицу «тренер → расчётный баланс» ДО включения.
3. Включить safeEnabled для Чангу (после «да» владельца).
4. Первый пересчёт в студии: фактические суммы вносятся коррекциями (историческую
   наличку частично забирали - расчётная цифра почти наверняка выше фактической).

## 7. Критерии качества

- Ровно один источник истины: приход не дублируется записями, расход только
  записями - цифра не может «разъехаться» от двойного учёта.
- Выплата из сейфа атомарна с TrainerPayment (одна транзакция).
- Изоляция студий; тренер видит только свой сейф; фича невидима при выключенном
  тумблере (страницы и API отвечают 403/404, пункт меню скрыт).
- `npm run build` + сценарный тест на scratch-БД (приход cash/абонемент,
  выплата из сейфа, изъятие, коррекция, фолбэк-атрибуция).
- Не трогать незакоммиченные CTWA-правки другой сессии (webhook, поля
  WhatsAppConversation): коммит - точечно своими файлами.
