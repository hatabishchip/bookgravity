# МЕТАПРОМТ: Допуск тренеров к «перевёрнутым позициям» (запрос Светланы, 16.07.2026)

Версия 1. Утверждён владельцем: (ожидает). Источник: два войса Светланы (админ Убуда) + 🔥 владельца на её втором варианте.

## Суть запроса
Инверсии (перевёрнутая позиция) - ответственная опция: у Андрея было два клиента подряд с паническими атаками в перевёрнутой позиции, неопытному тренеру (Рэдди) её вести нельзя. Лифтинг и «пальчики/стропы ног» могут все. Правило Светланы (одобрено владельцем): **если в классе тренер ИЛИ ассистент имеет допуск - бронирование инверсий открыто; нет допуска - опция закрыта.** Добавили Андрея ассистентом - опция открылась автоматически.

## Решения владельца (кнопками, 16.07)
- Правило «тренер ИЛИ ассистент с допуском» - подтверждено. Скрываем опцию и у клиента в виджете, и у персонала в развороте брони.
- Допуски раздаёт Света сама через админку (нужен переключатель в настройках тренера).
- Чангу не трогаем (услуга там выключена; механика заработает автоматически, когда включат).
- Лифтинг/пальчики - без допусков.

## Реализация
1. **Схема** (миграция prod Turso + prisma):
   - `Trainer.permInvertedPositions Boolean @default(false)` - допуск (рядом с permBookAnyClass/permManageBookings).
   - `AdditionalService.requiresInversionClearance Boolean @default(false)` - пометка «услуга требует допуска» (гибко: Света сможет пометить любую услугу, не хардкодим имя «Inverted position»).
   - Данные: услугам «Inverted position(s)» (Убуд активная, Чангу выключенная) проставить requiresInversionClearance=true. Допуски тренерам НЕ проставляем - Света сама.
2. **Хелпер** `slotAllowsInversions(slot)`: тренер?.permInvertedPositions || ассистент?.permInvertedPositions.
3. **Клиентский виджет** (BookingWidget + API слотов): услуга с requiresInversionClearance показывается/доступна только для слота, где slotAllowsInversions=true. При смене выбранного слота выбранная недоступная услуга сбрасывается.
4. **Сервер-валидация** POST /api/bookings: serviceIds с requiresInversionClearance отклоняются (400, понятный текст), если слот без допуска - защита от обхода UI.
5. **Staff**: API добавления услуги на бронь (bookings/[id]/services) - тот же гейт; в развороте брони услуга скрыта для класса без допуска.
6. **Админка**:
   - Настройки тренера: переключатель «Допуск: перевёрнутые позиции» (видит админ студии; UK-ключи).
   - Prices & Services: чекбокс «Требует допуска тренера (инверсии)» на услуге.
7. **E2E на проде**: слот Redi без ассистента - услуги нет в виджете и в развороте; добавить ассистентом тренера с допуском - услуга появилась; POST с обходом - 400. Тестовые данные вычистить.
8. **Сообщить Светлане** (текст на отдельное одобрение владельца): готово, как включить допуски.

## Критерии готовности
- Флаги в БД + UI-переключатели работают (скриншот).
- Гейт в трёх местах (виджет, POST-валидация, staff-API) проверен фактом.
- Убуд: услуга «Inverted position» помечена «требует допуска»; после раздачи допусков Светой всё открывается/закрывается составом слота.
