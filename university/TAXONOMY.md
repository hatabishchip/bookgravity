# Taxonomy

> **НАСТРОИТЬ ПРИ ИНИЦИАЛИЗАЦИИ ПРОЕКТА.** Этот файл — дерево тем, по которым строятся `syntheses/`. Каждый лист → один файл синтеза. При появлении нового листа — ДО создания синтеза — лист добавляется сюда с обоснованием, почему это отдельная тема, а не подраздел существующей.

## Пример структуры

```
<корневая-тема-1>/
├── <подтема-1.1>/       — описание
├── <подтема-1.2>/       — описание
└── <подтема-1.3>/       — описание

<корневая-тема-2>/
├── <подтема-2.1>/
└── <подтема-2.2>/
```

## Правила расширения таксономии

- Новый лист = новая тема, которую нельзя раскрыть в рамках существующих синтезов без потери ясности.
- Добавление листа фиксируется коммитом с сообщением `taxonomy: add <path>` и кратким обоснованием.
- Удаление/мердж листа допустим, если после 20+ источников тема не набрала 5 claim'ов — вероятно, это был не самостоятельный узел.
