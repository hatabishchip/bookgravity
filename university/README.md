# University — Knowledge Base for bookgravity

Цель: **построить и развивать надёжную онлайн-систему бронирования для студии растяжки Gravity Stretching (Чангу, Бали)**. Все знания здесь подкреплены первоисточниками и пригодны для последующего сравнения с реальным поведением продукта/кода.

Это не свалка ссылок. Это рабочая база знаний по правилам, описанным в [METHODOLOGY.md](METHODOLOGY.md).

## Структура

| Папка                | Что внутри                                                                                                                                   |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `sources/`           | Первоисточники «как есть» — HTML/PDF/транскрипты. Один источник = одна папка `sources/<src-id>/` с `meta.yaml`, `original.*`, `conspect.md`. |
| `notes/`             | Атомарные заметки (Zettelkasten): одна идея = один файл. Связаны перекрёстными ссылками.                                                     |
| `claims/`            | Проверяемые утверждения. Каждое — с доказательной базой и импликацией для проекта.                                                           |
| `syntheses/`         | Сводные документы по темам таксономии. Максимум 3000 слов на тему. Перезаписываются при появлении новой информации.                          |
| `contradictions/`    | Зафиксированные противоречия между источниками/утверждениями, ждущие разрешения.                                                             |
| `verdicts/`          | Итоговые выводы — что работает, что нет, почему.                                                                                             |
| `experiments/`       | Код + результаты собственных экспериментов для валидации claim'ов.                                                                           |
| `graph/edges.jsonl`  | Машиночитаемый граф связей (source → note → claim → synthesis).                                                                              |
| `queue/inbox.md`     | Куда пользователь кидает новые ссылки на обработку.                                                                                          |
| `queue/to-follow.md` | Внутренняя BFS-очередь ссылок, найденных внутри уже обработанных статей.                                                                     |
| `queue/escalation.md`| Вопросы к пользователю, на которые Claude не может ответить сам.                                                                             |
| `_templates/`        | Шаблоны для новых источников, заметок, утверждений, синтезов.                                                                                |
| `TAXONOMY.md`        | Что в скоупе, что вне — и дерево тем для синтезов.                                                                                           |
| `INDEX.md`           | Мастер-индекс всех источников/заметок/утверждений с статусами.                                                                               |

## Как пользоваться (оператору)

1. Кинь ссылку в [queue/inbox.md](queue/inbox.md) или просто напиши в чат.
2. Claude обрабатывает по пайплайну из METHODOLOGY.md §4: fetch → classify → score → conspect → atomize → recurse.
3. Периодически спрашиваешь «что у нас по теме X» — Claude читает `syntheses/` и `claims/`, сверяет с кодом/стратегией, выдаёт расхождения (по процедуре §14).

## Отношение к существующим документам проекта

Университет **не заменяет** существующую проектную документацию. Если у проекта уже есть `project-docs/`, `research/` или аналог — эти папки обрабатываются как seed-источники (см. [INDEX.md](INDEX.md) → Pre-existing internal sources) и добавляют поверх новое знание с внешних ссылок.

Если новое знание из Университета противоречит существующему — это оформляется как `contradiction`, а не тихая перезапись.
